# 基于 Dify + DeepSeek 的游戏租号售前智能客服

## 项目概述

这是一个面向游戏租号平台售前场景的 AI 智能客服项目。项目重点不是单纯做问答，而是围绕真实业务链路，把用户咨询、商品筛选、知识问答、导购推荐和售后分流整合成一套可控、可部署、可演示的系统。

项目展示页入口：
`http://101.43.56.2:8888`

在线体验通过展示页中的“立即体验”按钮进入。

## 技术栈

| 组件 | 技术选型 |
|------|----------|
| 工作流平台 | Dify Advanced Chat |
| 意图识别模型 | DeepSeek V4 Flash |
| 回答生成模型 | DeepSeek V4 Pro |
| 重排序模型 | Qwen3-Rerank |
| 知识库 | Dify RAG Pipeline |
| 后端网关 | FastAPI |
| 缓存层 | Redis |
| 外部能力 | HTTP API（导购检索） |
| 部署环境 | Ubuntu 22.04 + Nginx |

## 系统架构

```text
用户输入
  -> FastAPI 网关
  -> Redis 缓存代理
  -> Dify 工作流
      -> 对话补全
      -> 意图分流
      -> RAG 知识问答
      -> 导购推荐
      -> 异常兜底 / 转人工
```

## 核心设计

### 1. 多意图分流

系统先进行意图识别，再进入对应业务链路，避免一个模型同时承担分类、检索、生成和异常处理。当前覆盖的核心方向包括：

- 咨询问答
- 商品导购
- 订单查询
- 售后引导
- 打招呼 / 结束语 / 异常辱骂场景

### 2. 对话补全预处理

在正式分流前，先对用户输入做轻量补全，处理省略、指代和预算变化等常见表达问题。这样可以提升后续意图判断和导购参数提取的稳定性。

### 3. RAG 问答与导购链路拆分

知识类问题走知识库检索和重排，导购类问题走检索接口与筛选逻辑，避免“该查数据的问题被模型自由发挥”。

### 4. Redis 缓存与并发保护

高频重复问题优先命中缓存，并通过互斥锁避免短时间内重复打穿后端。这样既降低了大模型调用成本，也提升了线上响应稳定性。

### 5. 多层兜底机制

系统针对知识库无结果、导购无匹配、接口慢响应、模型异常等情况设计了兜底回复逻辑，保证对话不会轻易中断。

## 工程亮点

- 使用 Structured Output 约束意图分类和参数提取，降低解析错误率
- 使用重排序模型提升知识库问答命中质量
- 对 DeepSeek 输出做清洗，移除不应暴露给用户的思考标签
- 将不确定性的模型能力和确定性的业务逻辑拆层处理，增强整体可控性
- 完整补齐了展示页、线上体验入口和可直接演示的部署结果，适合作为作品集项目展示
