# 🐹 囤囤鼠 AI 智能客服 (TunTun AI Agent)

基于 Dify 工作流 + FastAPI 网关的电商智能客服系统，支持多意图分流、RAG 知识库问答、商品导购推荐和 Redis 缓存加速。

## 系统架构

```
用户请求
  │
  ▼
FastAPI 网关 (guide.py :8002)
  │
  ▼
Redis 缓存代理 (cache_proxy.py :8005)
  │  ├─ 命中缓存 → 秒回（TTL 1小时）
  │  └─ 未命中 ↓
  ▼
Dify 工作流引擎 (:81 Docker)
  │
  ├── 意图分流（deepseek-v4-flash）
  │     ├── CONSULT → RAG 知识库问答（LLM2: deepseek-v4-pro）
  │     ├── SHOPPING → 商品导购推荐（上游 API + 皮肤关键词筛选）
  │     └── HUMAN → 转人工
  │
  └── 查询改写（LLM5）
        ├── 5轮对话记忆
        ├── 省略句补全
        ├── 价格升降档逻辑（×0.75 / ×1.5）
        └── 下游正则清洗 think 标签
```

## 核心功能

### 1. 智能意图分流

使用 deepseek-v4-flash 快速分类用户意图，路由至对应处理分支：

- **CONSULT**：知识咨询 → RAG 检索增强生成
- **SHOPPING**：商品导购 → 按预算 / 皮肤关键词推荐账号
- **HUMAN**：超出知识库范围 → 转人工客服

### 2. RAG 知识库问答

- 混合检索 + qwen3-rerank 重排序
- Score 阈值 0.6（宁可漏答转人工，不编造答案）
- 普通 `###` 分块（解决 Q&A 分段导致的关键词丢失幻觉）
- 23 条测试验证：核心业务 8/8 命中（Score 0.76–0.98），知识库外 5 类 100% 正确转人工

### 3. 商品导购推荐

- F12 抓包逆向上游商品 API（参数 `skin_list1` / `skin_way1` + `page_size`）
- 支持按**皮肤名称**（信条、暗星等）+ **价格区间**双重筛选
- 无结果时输出友好提示

### 4. Redis 缓存加速

- `cache_proxy.py` 作为 Dify API 前置缓存代理
- 首次请求调用 Dify 并存入 Redis（TTL 1 小时），重复问题秒回
- 转人工回答不缓存（避免误导）
- 管理接口：`GET /cache/stats`（统计）、`DELETE /cache/clear`（清空）

### 5. 查询改写引擎

- 5 轮对话记忆上下文
- 省略句智能补全
- 价格升降档推理（×0.75 / ×1.5）
- 感谢 / 完整句原样透传
- 下游 code 节点正则清洗 `<think>` 标签

## 关键设计决策

1. **Score 阈值 0.6**：宁可漏答转人工，也不让 AI 编造答案，保障用户体验
2. **分块策略**：从 Q&A 分段改为普通 `###` 分块，解决关键词丢失导致的幻觉问题
3. **意图分流用 flash 模型**：快速 + 省 token，重活交给 pro 模型
4. **转人工不缓存**：避免缓存层拦截本应转人工的请求
5. **LLM2 定义类边界规则**：约束"满体满负"类轻度幻觉
