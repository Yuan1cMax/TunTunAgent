# 某电商公司 AI 智能客服

基于 Dify 工作流 + FastAPI 网关构建的售前智能客服系统，面向高频商品咨询场景，支持多意图分流、RAG 知识库问答、商品导购推荐和 Redis 缓存加速。

## 在线体验

- 前端入口页：`http://101.43.56.2:8888`
- 线上服务：`Dify + FastAPI + Redis + Nginx` 云端部署

## 项目亮点

- 多意图分流：先识别用户意图，再进入对应处理链路，避免一个模型同时承担分类与生成
- 知识库问答：RAG + `qwen3-rerank` 重排，提升售前问答命中率
- 商品导购：支持价格区间、关键词筛选与结果推荐，适合售前选品和账户筛选场景
- 缓存加速：对高频问题做 Redis 缓存，降低重复请求的 LLM 成本
- 并发控制：通过互斥锁避免重复查询和并发击穿
- 回答清洗：对模型输出做 `think` 标签清理，保持前台回答更干净

## 技术栈

| 层级 | 技术 |
|------|------|
| 工作流编排 | Dify Advanced Chat |
| 后端网关 | FastAPI (Python) |
| 缓存层 | Redis |
| 意图分流 | deepseek-v4-flash |
| 知识问答 | deepseek-v4-pro + qwen3-rerank |
| 导购接口 | FastAPI / HTTP API |
| 部署 | 腾讯云 Ubuntu 22.04 + Nginx |

## 架构概览

```text
用户请求
  -> FastAPI 网关 (guide.py :8002)
  -> Redis 缓存代理 (cache_proxy.py :8005)
  -> Dify 工作流
      -> 意图分流 (deepseek-v4-flash)
      -> CONSULT  -> RAG 知识库问答
      -> SHOPPING -> 商品导购推荐
      -> HUMAN    -> 转人工兜底
      -> QUERY REWRITE -> 查询改写
      -> think 标签清理
```

## 核心能力

### 1. 智能意图分流

使用 `deepseek-v4-flash` 对用户输入做快速分类，路由到不同分支：

- `GREETING`：打招呼
- `ABUSE`：辱骂兜底
- `END`：道别收尾
- `SHOPPING`：导购推荐
- `BUSINESS`：业务咨询

### 2. RAG 知识库问答

- 混合检索 + `qwen3-rerank`
- Score 阈值 `0.6`
- 采用原文分块策略，减少关键词丢失和幻觉回答
- 对超出知识库范围的问题可自动转人工

### 3. 商品导购推荐

- 支持价格区间筛选
- 支持皮肤/道具关键词筛选
- 支持导购 API 原生过滤后再做结果整理
- 查询不到时返回友好提示，而不是编造结果

### 4. Redis 缓存代理

- 首次请求访问 Dify，并将结果写入 Redis
- 高频重复问题直接命中缓存
- 转人工回答不缓存，避免错误复用
- 提供管理接口用于统计和清空缓存

### 5. 查询改写引擎

- 支持 5 轮对话记忆
- 补全省略词和指代
- 按价格梯度推导推荐范围
- 对无改写需求的原始输入保持不变

## 性能指标
